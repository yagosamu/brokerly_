# Renewals Sprint 20 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add renewal management with automatic daily detection, notifications, renewal execution, CRUD views, and Beat schedules.

**Architecture:** Create a standalone `renewals` app, keep renewal generation in service functions and Celery tasks, and expose tenant-scoped CBVs without templates. Beat schedules are managed by an idempotent command using `django-celery-beat`.

**Tech Stack:** Django 6, PostgreSQL, Celery, django-celery-beat, existing Brokerly tenant-aware models and role mixins.
